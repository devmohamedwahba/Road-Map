# 📌 Phase 15 — Interview Preparation (Start ~2 months before job hunt)

> System design (Phase 11) covers the technical content. This phase is about the *practice of performing* under interview conditions — DSA, live system design, and behavioral.

## 🧮 Data Structures & Algorithms

### Core Data Structures
- [ ] Arrays and strings
- [ ] Linked lists (singly, doubly)
- [ ] Stacks and queues
- [ ] Hash maps / hash sets
- [ ] Trees (binary, BST, tries)
- [ ] Heaps / priority queues
- [ ] Graphs (adjacency list/matrix)

### Core Algorithms
- [ ] Two pointers
- [ ] Sliding window
- [ ] Binary search (and its variants)
- [ ] BFS / DFS
- [ ] Recursion and backtracking
- [ ] Dynamic programming (1D, 2D, knapsack patterns)
- [ ] Greedy algorithms
- [ ] Sorting algorithms and complexity tradeoffs
- [ ] Union-Find (disjoint set)

### Practice Strategy
- [ ] 150-250 LeetCode problems (Blind 75 / NeoetCode 150 as a base list)
- [ ] Track patterns, not just problems — group by technique
- [ ] Time yourself (45 min per medium problem)
- [ ] Practice explaining your approach out loud before coding
- [ ] Mock interviews (pramp, interviewing.io, or with peers)

---

## 🏗️ System Design Interviews

- [ ] Practice the framework from Phase 11 out loud, on a whiteboard/doc, timed (35-45 min)
- [ ] Do 15-20 mock system design interviews (peers, mentors, or paid mock platforms)
- [ ] Record yourself and review — check pacing, clarity, tradeoff discussion
- [ ] Prepare 2-3 designs you can speak about in deep detail from your own projects
- [ ] Practice both "design X" and "here's our system, how would you improve it" formats

---

## 🗣️ Behavioral Interviews

- [ ] STAR method (Situation, Task, Action, Result)
- [ ] Prepare 8-10 stories covering: conflict, failure, leadership, ambiguity, mentoring, disagreement with a decision, tight deadline, technical tradeoff
- [ ] "Tell me about yourself" — 90 second version, tailored
- [ ] Questions to ask the interviewer (shows engagement)
- [ ] Research the company's engineering blog/tech stack before each interview

---

## 📄 Application Materials

- [ ] Resume tailored per role (ATS-friendly, quantified impact)
- [ ] LinkedIn profile aligned with resume
- [ ] Portfolio links ready (Phase 12 projects) with live demos if possible
- [ ] Elevator pitch for networking/referrals

## 🚀 Weekly Cadence (during active job search)

- [ ] 5-8 DSA problems/week
- [ ] 1-2 mock system design interviews/week
- [ ] 1 behavioral story refined/week
- [ ] Apply to 5-10 tailored roles/week
- [ ] Follow up on referrals and networking weekly

## 🧑‍💼 Senior Engineer Soft Skills

> These aren't tested in a single interview round, but they show up in behavioral questions and are what actually separate senior from mid-level on the job.

### Technical Leadership
- [ ] Writing design docs / RFCs (structure: context, problem, options, recommendation)
- [ ] Running technical reviews (code reviews, architecture reviews)
- [ ] Estimating project timelines (breaking down unknowns, buffers)
- [ ] Making and documenting technical decisions (ADRs — Architecture Decision Records)
- [ ] Saying "no" with data (pushing back on bad requirements)

### Communication
- [ ] Explaining technical tradeoffs to non-technical stakeholders
- [ ] Writing clear incident postmortems (blameless)
- [ ] Status updates that surface risks early
- [ ] Async communication (writing > meetings for distributed teams)

### Mentoring & Collaboration
- [ ] Giving constructive code review feedback
- [ ] Mentoring junior engineers (pair programming, guided questions)
- [ ] Onboarding new team members
- [ ] Cross-team collaboration (aligning on APIs, contracts, timelines)

### Ownership
- [ ] On-call rotation and incident response
- [ ] Monitoring and alerting ownership
- [ ] Technical debt tracking and prioritization
- [ ] Knowing when to refactor vs ship
